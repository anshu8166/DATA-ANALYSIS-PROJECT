# 🚕 Uber Multi-Service Operations Dashboard: Data-Driven Performance Analysis

### 📊 Project Overview

This project presents an **interactive Uber Operations Dashboard** built using detailed ride-sharing data. The dashboard performs a comparative analysis of key services (*Auto, Bike, Go Mini, Go Sedan, Premier Sedan, Uber XL*) and provides critical insights into **operational efficiency** and **profitability**.

The purpose is to provide **data-driven insights** to optimize resource allocation, reduce operational friction, and improve overall customer experience.

### 🔑 Key Operational Insights

* **Total Revenue & Bookings** 💰 per vehicle type.
* **Service Status** ✔️ (Completed, Cancelled, Incomplete rides).
* **Operational Friction:** Breakdown of **Cancellations by Driver vs. Customer** 🛑.
* **Geographical Hotspots** 📍: Top Pickup and Dropoff locations.
* **Temporal Trends:** Revenue and booking performance by **Year/Month**.
* **Service Quality:** Average Customer and Driver Ratings ⭐.

### ⭐ How to Use the Dashboard

1.  Open the [**Dashboard Preview/File Folder**](https://drive.google.com/drive/folders/19V5QJf8bUTIsfmZkbUU52rTRIwjaEGW2?usp=sharing) to view the project output.

2.  Use the **vehicle type selection** buttons to filter the data for specific services like 'Uber XL' or 'Auto'.

3.  Utilize the **Month/Quarter slicers** to explore time-based trends in revenue and bookings.

4.  Focus on the **Cancellation charts** to identify the most significant points of operational failure (Driver vs. Customer).

5.  Watch the demo video for a guided tour of the dashboard's features.
